# 自助提报自动化脚本使用说明

## 📌 功能概述

本脚本实现了以下核心功能：

### 1️⃣ 获取会话ID
- 调用API接口1 (`/api/ess/hroAttendance/ai-self-fill/start`)
- 获取响应中的 `conversationId`

### 2️⃣ 批量发送消息
- 从Excel文件的【输入】列读取消息内容
- 调用API接口2 (`/api/ess/hroAttendance/ai-self-fill/chat`)
- 传递参数：
  - `content`: 从Excel【输入】列获取
  - `conversationId`: 从接口1获取
  - `periodStartDate`: 可配置
  - `periodEndDate`: 可配置
  - `X-AUTH`: 可配置（请求头）

### 3️⃣ 结果回写
- 将API响应中的 `content` 写入Excel的【实际输出】列

## 🚀 快速开始

### 步骤1：安装依赖

```bash
# 方式1：双击运行（推荐）
使用uv安装.bat

# 方式2：使用启动器
启动脚本.bat → 选择 [2] 安装/更新依赖

# 方式3：命令行运行
uv sync
```

### 步骤2：配置参数

编辑 `config.json` 文件：

```json
{
    "x_auth": "AFC0937794E21C554173E597B33640BA",
    "period_start_date": "2025-12-01",
    "period_end_date": "2025-12-31",
    "excel_path": "e:\\同道\\HCM\\自助提报自动化\\自助填报会话.xlsx",
    "user_id": "346568",
    "collection_group_record_id": "166",
    "input_column": "输入",
    "output_column": "实际输出"
}
```

### 步骤3：准备Excel文件

Excel文件格式示例：

| 输入 | 实际输出 |
|------|----------|
| 全勤，未请假 | （API响应） |
| 请假1天 | （API响应） |
| 加班2小时 | （API响应） |

- **输入**列：必须存在，填写要发送的消息，多轮会话拼接“say”
- **实际输出**列：如果不存在会自动创建
- 每行共用一个会话id

### 步骤4：运行脚本

```bash
# 方式1：双击运行（推荐）
使用uv运行.bat

# 方式2：使用启动器
启动脚本.bat → 选择 [1] 运行脚本

# 方式3：命令行运行
uv run python run_with_config.py
```

## ⚠️ 注意事项

### 1. 数据安全
- ✅ 运行前请备份Excel文件
- ✅ 确保配置文件中的token安全

### 2. 网络要求
- ✅ 确保网络连接正常
- ✅ 能够访问API接口地址
- ✅ X-AUTH token有效且未过期

### 3. 数据格式
- ✅ 输入列不应有空值（空值会自动跳过）
- ✅ 日期格式为 YYYY-MM-DD
- ✅ Excel文件不要在打开状态下运行脚本

### 4. 会话管理
- ✅ 所有行使用同一个会话ID处理
- ✅ 支持多轮对话

### 5. 错误处理
- ✅ 单行失败不影响其他行
- ✅ 详细错误日志输出
- ✅ 网络超时时间为30秒

## ❓ 常见问题

### Q1: 提示找不到模块？
**A:** 运行 `安装依赖.bat` 或 `pip install -r requirements.txt`

### Q2: Excel文件被占用？
**A:** 关闭Excel文件后再运行脚本

### Q3: API调用失败？
**A:** 检查以下几项：
- 网络连接是否正常
- X-AUTH token是否有效
- 日期格式是否正确（YYYY-MM-DD）
- API接口地址是否可访问

### Q4: 如何修改配置？
**A:** 编辑 `config.json` 文件，修改相应参数后保存

### Q5: 如何查看详细日志？
**A:** 运行脚本时，控制台会实时显示详细的处理过程和日志

### Q6: 可以处理多少行数据？
**A:** 理论上没有限制，但建议单次处理不超过1000行，避免会话超时

### Q7: 失败后需要重新运行吗？
**A:** 可以重新运行，已处理的行会被覆盖。建议先查看日志确定失败原因
